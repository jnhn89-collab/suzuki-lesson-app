# 스즈키 레슨 성과보고서 디자인 구상

## 목적

선생님이 이번 분기 또는 이번 학기의 평가를 한 번 입력하고, 학부모는 모바일에서 바로 읽을 수 있는 성과보고서를 받는다. 보고서는 아이를 평가한다는 느낌보다, 해당 기간 동안 무엇이 좋아졌고 다음 기간에 무엇을 연습하면 되는지 알려주는 학기 리포트에 가까워야 한다.

## 사용자 흐름

1. 선생님이 평가 기간, 기본 정보, 누적 루브릭 점수를 입력한다.
2. 잘된 점, 성장 포인트, 가정 연습 문장을 프리셋으로 빠르게 선택한다.
3. 필요한 문장만 직접 다듬는다.
4. 오른쪽 미리보기에서 학부모용 보고서를 확인한다.
5. 인쇄/PDF 또는 이후 Apps Script 저장 API로 연결한다.

## 문체 원칙

- AI가 요약한 듯한 설명체를 피한다.
- "학생은 ~ 단계입니다"보다 "이번 기간 동안 ~가 좋아졌습니다"처럼 기간 관찰 문장으로 쓴다.
- 학부모에게 부담을 주는 문장보다 집에서 바로 도와줄 수 있는 행동을 제안한다.
- 비교, 평가, 지표 설명 문구는 최소화한다.
- 점수는 보조 정보로 두고, 본문은 관찰과 연습 처방 중심으로 구성한다.

## 화면 구조

### 선생님 입력 패널

- 모바일에서도 한 컬럼으로 자연스럽게 내려온다.
- 입력은 `기본 정보 -> 점수 -> 프리셋 -> 직접 수정 -> 연습 처방` 순서다.
- 프리셋은 문장 버튼을 누르면 textarea에 누적된다.
- 프리셋 카테고리:
  - 잘된 점
  - 성장 포인트
  - 가정 연습 도움말
  - 교정 포인트
  - 칭찬 문장

### 보고서 패널

- 모바일에서는 보고서가 먼저 읽히도록 카드형 단일 컬럼이다.
- 학부모 전달용 모바일 HTML에서는 입력 패널 없이 보고서만 읽히는 `share-mode`를 제공한다.
- 데스크톱에서는 입력 패널과 보고서 미리보기를 나란히 배치한다.
- 보고서는 표지 느낌을 과하게 만들지 않고, 실제 학부모가 읽는 리포트처럼 조밀하고 정돈된 레이아웃을 사용한다.

## 타이포그래피

- 본문: Pretendard, SUIT, Noto Sans KR, Apple SD Gothic Neo fallback
- 제목: Paperlogy, Pretendard fallback
- 웹폰트 로딩이 실패해도 시스템 한글 폰트로 자연스럽게 보이게 한다.
- 모바일 본문은 15-16px, 줄간격 1.65 이상을 유지한다.
- 보고서 본문은 너무 얇은 weight를 쓰지 않는다.

## 색상

- 선생님 입력 패널은 업무용 느낌의 slate/blue를 사용한다.
- 학부모 보고서는 따뜻한 off-white 바탕에 ink/blue/green 보조색을 사용한다.
- 점수 색은 과하게 빨강/초록으로 평가처럼 보이지 않게 한다.

## 모바일 기준

- 390px 폭에서 버튼 텍스트가 넘치지 않아야 한다.
- 프리셋 버튼은 두 줄까지 허용한다.
- 보고서 주요 내용은 첫 화면에서 학생 이름, 평가 기간, 현재 곡, 기간 요약이 보인다.
- 인쇄 버튼은 모바일에서도 상단 고정이 아니라 입력 패널 하단에 둔다.
